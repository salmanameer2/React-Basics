Learn React.js with Vite

Complete guide from installation to deployment. Learn how to build, version control with Git, and deploy your React apps on Netlify.

Get Started

What is React.js?

React is a popular JavaScript library for building user interfaces, developed by Facebook. It allows developers to create reusable UI components and manage application state efficiently.

Component-Based

React applications are built using components - reusable, self-contained pieces of code that manage their own state and rendering.

Declarative

React uses a declarative paradigm that makes it easier to reason about your application and helps create interactive UIs.

Virtual DOM

React uses a virtual DOM to optimize rendering, only updating the parts of the DOM that have actually changed.

What is Vite?

Vite (French word for "fast", pronounced /vit/) is a next-generation frontend build tool that significantly improves the development experience for modern web projects.

Lightning Fast

Vite provides near-instant server start and hot module replacement (HMR) that stays fast regardless of app size.

Optimized Build

Vite uses Rollup for production builds, pre-configured to output highly optimized static assets.

Rich Features

Out-of-the-box support for TypeScript, JSX, CSS, and more. Highly extensible via its plugin API.

Installation & Setup

Setting up a React project with Vite is straightforward. Follow these steps to get started:

Step 1: Create a new Vite project

# Using npm
npm create vite@latest my-react-app -- --template react

# Using yarn
yarn create vite my-react-app --template react

# Using pnpm
pnpm create vite my-react-app --template react

Step 4: Start the development server

# Using npm
npm run dev

# Using yarn
yarn dev

# Using pnpm
pnpm dev

React Usage & Concepts

Once your React + Vite project is set up, you can start building components and applications. Here are key concepts to understand:

Components

React apps are built from components - reusable pieces of UI. Components can be functional or class-based.

State Management

React components can have state that determines their behavior and rendering. Use useState hook for functional components.

Props

Props (properties) allow data to be passed from parent to child components, making components reusable and configurable.

Git Version Control & GitHub

Version control is essential for tracking changes, collaborating with others, and deploying your applications. Git is the most popular version control system, and GitHub is a platform for hosting Git repositories.

Why Use Git?

Git helps you track changes to your code, collaborate with other developers, and maintain different versions of your project. It's essential for professional development workflows.

Complete Git & GitHub Setup Process

1. Initialize Git
2. Create .gitignore
3. Stage & Commit
4. Connect to GitHub
5. Push to Remote

Step-by-Step Guide

  1. Install Git (if not already installed)

    Download and install Git from git-scm.com. Verify installation with:

    git --version

  2. Configure Git with your credentials

    Set up your name and email for commits:

    # Set your username
    git config --global user.name "Your Name"

    # Set your email
    git config --global user.email "youremail@example.com"

  3. Initialize Git in your React project

    Navigate to your project folder and initialize Git:

    # Navigate to your project directory
    cd my-react-app

    # Initialize a new Git repository
    git init

  4. Create a .gitignore file

    Create a file named .gitignore in your project root to exclude unnecessary files:

    # Dependency directories
    node_modules/
    .pnpm-store/

    # Build output
    dist/
    build/

    # Environment variables
    .env
    .env.local
    .env.development.local
    .env.test.local
    .env.production.local

    # Logs
    npm-debug.log*
    yarn-debug.log*
    yarn-error.log*

    # Editor directories and files
    .vscode/
    .idea/
    *.suo
    *.ntvs*
    *.njsproj
    *.sln
    *.sw?

  5. Stage and commit your files

    Add your files to the staging area and create your first commit:

    # Add all files to staging area
    git add .

    # Or add specific files
    git add index.html src/ package.json

    # Commit with a descriptive message
    git commit -m "Initial commit: React app with Vite"

  6. Create a GitHub repository

    Go to GitHub.com, log in, and create a new repository. Don't initialize it with README, .gitignore, or license if you've already done these locally.

  7. Connect local repository to GitHub

    Link your local Git repository to the remote GitHub repository:

    # Add remote repository (replace with your URL)
    git remote add origin https://github.com/your-username/your-repo-name.git

    # Verify the remote URL
    git remote -v

  8. Push your code to GitHub

    Upload your local commits to the GitHub repository:

    # Push to the main branch
    git push -u origin main

    # If your default branch is master, use:
    git push -u origin master

    Success!

    Your React project is now on GitHub! You can visit your repository URL to see the code.

Common Git Commands Cheat Sheet

Branch Management

git branch # List branches

git checkout branch # Switch branch
git merge branch # Merge branch

History & Changes

git status # Check status
git log # View commit history

Undo & Recovery

git stash # Temporarily stash changes

Deploy React App to Netlify

Netlify is a popular platform for deploying static websites and web applications. It offers continuous deployment from Git, free SSL, CDN, and more. Here's how to deploy your React + Vite app to Netlify.

Why Netlify?

Netlify provides a simple, fast, and reliable way to deploy web projects with automatic builds from Git, free hosting for personal projects, and powerful features like form handling, serverless functions, and instant rollbacks.

Deployment Options

Netlify

Easiest deployment with drag & drop, Git integration, and free tier

Vercel

Created by Next.js team, optimized for React, excellent performance

GitHub Pages

Free hosting for static sites directly from GitHub repositories

Netlify Deployment Process

1. Build Project
2. Create Netlify Account
3. Connect Git Repo
4. Configure Build Settings
5. Deploy!

Method 1: Deploy via Netlify CLI (Command Line)

  1. Build your React project for production

    First, create an optimized production build of your app:

    # Build the project (creates a 'dist' folder)
    npm run build

    # Preview the production build locally
    npm run preview

  2. Install Netlify CLI globally

    Install the Netlify command line interface:

    # Using npm
    npm install -g netlify-cli

    # Using yarn
    yarn global add netlify-cli

  3. Login to Netlify from CLI

    Authenticate with your Netlify account:

    # Login to Netlify
    netlify login

    This will open a browser window for authentication. Follow the prompts to authorize.

  4. Initialize and deploy your site

    Deploy your built application to Netlify:

    # Initialize a new Netlify site
    netlify init

    # Deploy to Netlify (from the dist folder)
    netlify deploy --prod --dir=dist

    Follow the prompts to create a new site or link to an existing one.

Method 2: Deploy via Netlify Web UI (Recommended for Beginners)

  1. Push your code to GitHub

    Ensure your React project is on GitHub (as shown in the Git section).

  2. Create a Netlify account

    Go to netlify.com and sign up (you can use your GitHub account for easy login).

  3. Add a new site from Git

    Click "Add new site" → "Import an existing project" → Connect to GitHub.

  4. Select your repository

    Choose your React project repository from the list.

  5. Configure build settings

    Netlify will auto-detect Vite projects, but verify these settings:

    # Build command
    npm run build

    # Publish directory
    dist

    # Environment variables (if needed)
    VITE_API_URL=https://api.example.com

  6. Deploy site

    Click "Deploy site". Netlify will build and deploy your application.

    Deployment Complete!

    Your React app is now live! Netlify will provide a URL (like https://your-site-name.netlify.app).

Method 3: Drag & Drop Deployment (Simplest)

  1. Build your React project

    npm run build

  2. Locate the dist folder

    After building, you'll have a dist folder in your project.

  3. Drag & drop to Netlify

    Go to Netlify Dashboard → Sites → Drag and drop your dist folder.

    Netlify will upload and deploy your site instantly!

Important Configuration for Vite Projects

For single-page applications (SPAs) like React apps, you need to configure redirects for proper routing:

Create a file named _redirects in your public folder with this content:

# Single Page App Routing
/* /index.html 200

Or create a netlify.toml file in your project root:

[[redirects]]
from = "/*"
to = "/index.html"
status = 200

Post-Deployment: Custom Domain & Environment Variables

Custom Domain

Go to Site Settings → Domain Management → Add custom domain. Follow Netlify's instructions to configure your DNS.

Environment Variables

In Site Settings → Environment Variables, add variables used in your app (prefixed with VITE_ for Vite).

Continuous Deployment

Every push to your connected Git branch triggers an automatic rebuild and deployment.
